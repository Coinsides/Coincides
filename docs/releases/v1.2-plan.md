# Coincides v1.2 — 版本规划

> **主题**：AI 交互重构 + Goal Manager 完整 UI + 新用户引导 + i18n
> **前置版本**：v1.1（UX 痛点修复 + 技术债偿还 + 数据模型升级）
> **目标**：重写 System Prompt 使 AI 符合 MWF 哲学，完善 Goal Manager 为完整树形 UI，添加新用户引导流程，支持中英双语。

---

## 一、核心概念对齐

本版本的核心目标是让 AI（Mr. Zero）从"知识问答机器"转变为"脚手架搭建者"——用户不需要理解底层的学习科学，AI 在内部综合策略来服务用户。

- **System Prompt 重写**：当前 system-prompt.ts 中的「Study Plan Creation Protocol」存在根本性反模式——把 7 种正交的学习模式当成互斥菜单让用户选择，把用户推入他没有能力做出的决策（详见第 1 次会议记录 §2）。重写后 AI 应遵循脚手架理论（Scaffolding Theory）：拆解任务、分级建议、逐步撤除支撑。
- **设计宪法落地**：三条不可违反的原则必须硬编码进 System Prompt 和 Agent 行为逻辑中：
  1. 不替用户做决定——AI 只拆解、只建议、只执行
  2. 不监控用户——不追踪用时、不判断精力、不主动生成用户没要求的东西
  3. 不制造挫败感——不锁死时间、不自动回顾失败、跳过任务零惩罚
- **Goal Manager 完整 UI**：v1.1 只做了一层缩进展示和基础添加功能，v1.2 要实现完整的树形展示 + 拖拽排序 + 阶段管理。
- **Agent 工具升级**：Agent 需要理解 Goal 层级关系，能够自动将大目标拆解为子目标和 Task。

> v1.2 完成后，用户可以用自然语言告诉 AI 学习目标，AI 通过 Proposal 机制输出结构化计划，用户审批后执行。

## 三、Step 拆分

### Step 1：System Prompt 重写（AI-1 + AI-2）

**目标**：彻底重写 `system-prompt.ts`，从「学习科学菜单」转为「MWF 脚手架助手」。

**工作内容**：
- 移除「Study Plan Creation Protocol」（第 53-59 行的 7 步模板选择流程）
- 移除 `get_study_templates` 工具调用（7 种学习模式不再暴露给用户）
- 重写 System Prompt 核心指令：
  - AI 角色定义：你是学习脚手架搭建者，不是学习科学顾问
  - 设计宪法三条原则作为 HARD RULES 写入
  - 交互准则：简洁、工作流导向、不答非所问
  - MWF 三级分层逻辑：Must > Recommended > Optional
  - Recommended/Optional 必须标注为哪个 Must 服务
- 保留并优化现有有效部分（Proposal 机制、Document-Based Card Generation 等）

**验收标准**：
- [ ] 用户问「你将如何创建我的学习计划？」→ AI 回答工作流步骤（不是学习方法菜单）
- [ ] AI 不再主动调用 `get_study_templates`
- [ ] System Prompt 中包含设计宪法三条原则
- [ ] AI 不主动追踪用时、不判断精力、不自动生成报告

---

### Step 2：Agent 工具升级（AI-3 + AI-4 + AI-5）

**目标**：Agent 理解 Goal 层级，能自动拆解目标为子目标和 Task，Proposal 自动标注优先级。

**工作内容**：
- 升级 `list_goals` 工具：返回层级结构（包含子 Goal 和关联 Task）
- 新增 `create_sub_goal` 工具：在指定 Goal 下创建子目标
- 升级 `create_proposal` 工具：
  - `study_plan` 类型 Proposal 自动标注每个 Task 的 priority
  - 新增 `goal_breakdown` Proposal 类型：大目标 → 子目标 → Task 的完整拆解
  - Proposal 中每个 Recommended / Optional 标注为哪个 Must 服务
- 升级 `create_task` 工具：支持同时生成 checklist 子步骤
- System Prompt 新增「Goal Breakdown Protocol」引导 Agent 正确使用新工具

**验收标准**：
- [ ] Agent 可接收用户的大目标，通过 Proposal 输出子目标 + Task 拆解方案
- [ ] 每个 Proposal 中的 Task 都标注了 Must / Recommended / Optional
- [ ] Recommended / Optional 标注为哪个 Must 服务
- [ ] Agent 创建 Task 时可包含 checklist 子步骤
- [ ] 用户审批 Proposal 后，Goal 层级和 Task 正确写入数据库

---

### Step 3：Goal Manager 完整 UI（GM-1 + GM-2 + GM-3）

**目标**：Goals 页面从简单缩进列表升级为完整的树形管理界面。

**工作内容**：
- 树形展示组件：
  - 可展开/折叠的多层树状结构
  - 每层缩进 + 连接线可视化
  - Goal 节点显示：名称、进度条（子 Task 完成百分比）、子目标数量、Task 数量
- 拖拽排序：
  - 同层级 Goal 拖拽调整顺序
  - 跨层级拖拽（改变父子关系）
  - 拖拽视觉反馈（插入线、高亮目标位置）
- 阶段管理：
  - Goal 进度自动计算（基于子 Task 完成状态）
  - 批量操作：批量完成、批量删除子 Task
  - Goal 状态标记（进行中 / 已完成 / 暂停）
- 后端 API 补充：
  - `PUT /api/goals/reorder` — 批量更新顺序和父子关系
  - `GET /api/goals/:id/progress` — 获取 Goal 及子 Goal 的进度统计

**验收标准**：
- [ ] Goals 页面显示完整树形结构（支持 3+ 层嵌套）
- [ ] 可拖拽调整 Goal 顺序和层级
- [ ] Goal 节点显示子 Task 完成进度
- [ ] 批量操作正常工作
- [ ] 性能：100 个 Goal 的树形渲染 < 200ms

---

### Step 4：Goal-日历联动（GM-4）

**目标**：日历视图中 Task 按所属 Goal 分组着色，用户一眼看到哪些任务属于哪个目标。

**工作内容**：
- Goal 颜色系统：每个顶级 Goal 自动分配一个主题色（可手动修改）
- 日历渲染升级：
  - Task 时间块按 Goal 主题色着色
  - 日历侧栏新增 Goal 图例（颜色 + 名称）
  - 支持按 Goal 过滤日历视图
- 后端：Task 查询时携带 Goal 颜色信息
- Schema 变更（通过 migration）：
  ```sql
  ALTER TABLE goals ADD COLUMN color TEXT DEFAULT '#6366f1';
  ```

**验收标准**：
- [ ] 有 Goal 的 Task 在日历中按 Goal 颜色显示
- [ ] 日历侧栏显示 Goal 图例
- [ ] 可按 Goal 过滤日历视图
- [ ] 无 Goal 的 Task 正常显示（使用默认色）

---

### Step 5：新用户引导流程（OB-1 + OB-2 + OB-3）

**目标**：新用户注册后有清晰的引导路径，不会感到迷茫。

**工作内容**：
- 引导状态追踪：
  - `users` 表新增 `onboarding_completed` 字段（Boolean）
  - 前端 `authStore` 追踪引导状态
- 引导步骤 UI：
  - Step 1：注册成功后弹出欢迎页，AI 说「你在学什么？创建一门课程开始吧。」
  - Step 2：引导用户到 Courses 页面创建第一门课程
  - Step 3：课程创建后提示上传材料或在聊天栏描述学习目标
  - Step 4：引导用户打开 Agent 面板（Ctrl+J）与 AI 对话
  - 每步可跳过，跳过不惩罚
- 功能提示（Tooltip）：
  - 关键页面首次访问时显示简短提示气泡
  - 用户关闭后不再显示（localStorage 记录）
  - 不强制、不阻挡操作

**验收标准**：
- [ ] 新注册用户看到引导流程
- [ ] 每步可跳过，不影响使用
- [ ] 引导完成后标记 `onboarding_completed = true`，不再显示
- [ ] 已有用户不受影响

### Step 6：i18n 国际化框架（I18N-1 + I18N-2 + I18N-3）

**目标**：支持中英双语，为未来多语言扩展打基础。

**工作内容**：
- 选型：`react-i18next`（React 生态最成熟的 i18n 方案）
- 翻译文件结构：
  ```
  client/src/locales/
  ├── en/
  │   └── translation.json
  └── zh/
      └── translation.json
  ```
- 全局文案抽取：
  - 所有页面硬编码文案替换为 `t('key')` 调用
  - 按页面/组件分 namespace（避免单文件过大）
- Settings 语言切换：
  - Settings 页面新增 Language 选项（English / 中文）
  - 切换后即时生效，存入用户 settings
- Agent 语言适配：
  - System Prompt 根据用户语言设置追加语言指令
  - `{{language}}` 模板变量注入

**验收标准**：
- [ ] 所有 UI 文案支持中英双语
- [ ] Settings 切换语言后即时生效
- [ ] Agent 回复语言与用户设置一致
- [ ] 新增翻译只需编辑 JSON 文件，不改代码

## 四、执行顺序

```
Step 1 (System Prompt 重写)
  ↓
Step 2 (Agent 工具升级) ← 依赖新 prompt 的指令框架
  ↓
Step 3 (Goal Manager UI) ← 独立 UI 工作，但需配合 Agent 新工具
  ↓
Step 4 (Goal-日历联动) ← 依赖 Goal 颜色字段
  ↓
Step 5 (新用户引导) ← 依赖 System Prompt 和 Agent 工具就绪
  ↓
Step 6 (i18n 国际化) ← 全局文案替换，放在功能开发后统一做
  ↓
Step 7 (Agent 报错) ← 灵活插入，依赖 Henry 复现
```

**执行顺序**：1 → 2 → 3 → 4 → 5 → 6 → 7

## 五、为后续版本铺路

v1.2 的 AI 交互重构和 Goal Manager 为后续版本打基础：

| v1.2 做的 | 为哪个版本铺路 | 后续怎么用 |
|---|---|---|
| System Prompt 重写 | v1.3 工作流引擎 | L1 入驻流需要 AI 按正确的脚手架流程引导用户 |
| Agent 目标拆解能力 | v1.3 工作流引擎 | 工作流引擎 = Agent 拆解 + 排期 + 执行的自动化 |
| Goal Manager 树形 UI | v1.3 工作流引擎 | 树形 Goal 是 L1 入驻流「大目标 → 阶段 → Task」的可视化 |
| Goal-日历联动 | v1.3 Time Block | Time Block 需要按 Goal 分配学习时段 |
| i18n 框架 | v1.4+ 多语言扩展 | 后续添加更多语言只需新增 JSON 文件 |
| 新用户引导 | v1.3 L1 入驻流 | 引导流程是 L1 入驻流的 UI 骨架 |
