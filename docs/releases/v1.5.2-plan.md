# Coincides v1.5.2 — 版本规划

> **主题**：Task-Block 关联完善：TB 选择器 + Hover 任务面板 + 未归类区分
> **前置版本**：v1.5.1（TB UX 优化）
> **目标**：让 Task 创建/编辑时可选关联 TB；TB 上 hover 弹出可交互的任务面板（可滚动、可编辑）；未归类任务在视觉上与已归类任务区分。

---

## 一、设计宪法守则

| 宪法条款 | v1.5.2 约束 |
|---|---|
| 不替用户做决定 | TB 选择器是可选的，不强制关联——用户可以跳过 |
| 不监控用户 | 不追踪"未归类率"，不提示"你有 N 个未归类任务" |
| 不制造挫败感 | 未归类任务只是视觉标记不同，不是"错误状态" |

---

## 二、需求条目

| ID | 模块 | 描述 | 复杂度 | 类型 |
|----|------|------|--------|------|
| TB-Q1 | Task 创建/编辑 | 创建/编辑 Task 时增加 TB 下拉选择器（可选） | 中 | 新增 |
| TB-Q2 | 周视图交互 | TB hover 300ms 后弹出侧边任务面板，可滚动、可编辑 | 中 | 新增 |
| TB-Q3 | 周视图视觉 | 未关联 TB 的 timed task 视觉区分（半透明/虚线边框） | 低 | 优化 |

---

## 三、Step 拆分

### Step 1：Task 创建/编辑时增加 TB 选择器（TB-Q1）

**变更**：
1. `shared/types/index.ts` — `CreateTaskRequest` 和 `UpdateTaskRequest` 增加 `time_block_id?: string | null`
2. `TaskModal.tsx` — 新增状态 `timeBlockId`，表单中增加 TB 下拉选择器：
   - 根据当前 date 获取当天的 TB 列表（从 timeBlockStore.weekData）
   - 显示为 `<select>`：默认 "No block"（空值） + 当天各 TB 选项（类型名 + 时间范围）
   - 创建/更新时传递 `time_block_id`
3. `TaskModal.tsx` — 编辑模式下预填现有 `time_block_id`

### Step 2：TB Hover 延展任务面板（TB-Q2）

**变更**：
1. `Calendar.tsx` — 新增 hover 状态：`hoveredBlock`（block + position）
   - TB 背景层增加 `onMouseEnter` / `onMouseLeave` 事件（300ms 延迟触发）
   - 面板显示在 TB 右侧（或左侧，取决于屏幕空间）
2. `Calendar.tsx` — 渲染 hover 面板：
   - 面板标题：TB 类型名 + 时间范围
   - 任务列表：按 must/recommended/optional 分组
   - 每个任务：checkbox（可切换完成状态）+ 标题 + 课程色条
   - 点击任务标题 → 打开 task-edit modal
   - 面板可滚动（max-height + overflow-y: auto）
3. `Calendar.module.css` — hover 面板样式：
   - fixed/absolute 定位，毛玻璃背景，圆角，阴影
   - 鼠标可移入面板（面板和 TB 构成一个连续的 hover 区域）
   - 退出延迟 200ms 防误关

### Step 3：未归类任务视觉区分（TB-Q3）

**变更**：
1. `Calendar.tsx` — timed task 渲染时检查 `task.time_block_id`：
   - 有 TB 关联：正常渲染（实线边框 + 完整不透明度）
   - 无 TB 关联：虚线左边框 + 略微降低不透明度（opacity: 0.7）
2. `Calendar.module.css` — `.taskUnassigned` 样式
