# Coincides v1.5.1 — 版本规划

> **主题**：Time Block UX 优化：Bug 修复 + 类型系统重构 + 视觉优化
> **前置版本**：v1.5（Time Block 重构：图层嵌套 + 编辑模式 + 任务关联）
> **目标**：修复框选创建 Bug；重构 TB 类型系统（type 即名字，去掉独立 label，开放自定义类型）；TB badge 改为 must/recommended/optional 分类计数；Day Detail 面板从侧边栏改为居中浮层。

---

## 一、问题背景

v1.5 完成了 Time Block 的核心重构，但实际使用中暴露出以下问题：

**Bug**：
- 框选区域后右键无法添加 Time Block——因为 `onMouseDown`（含右键触发）会先清空 `dragSelection`，导致 `contextmenu` 事件到达时选区已丢失

**设计问题**：
- Time Block 的 label 和 type 是冗余的——用户反馈 "time block 按类型分，类型就是它的名字"，不需要再给 TB 单独起名
- 类型系统封闭——只有 Study / Sleep / Custom 三个选项，没有添加自定义类型的途径；Custom 类型无法编辑描述性名称
- TB badge 只显示总任务数，用户想看到 must / recommended / optional 的分类计数
- Day Detail 面板作为右侧侧边栏（400px 宽），用户期望改为居中浮层页面

---

## 二、设计宪法守则（v1.5.1 特别提醒）

| 宪法条款 | v1.5.1 约束 |
|---|---|
| 不替用户做决定 | 自定义类型由用户手动添加，系统不自动推断或建议类型 |
| 不监控用户 | 类型使用频率不追踪、不排序 |
| 不制造挫败感 | 没有设置类型的 TB 保持 "custom" 默认，不提示"请先设置类型" |

## 四、Step 拆分

### Step 1：Bug 修复 + Day Detail 居中浮层（TB-P1 + TB-P5）

**目标**：修复框选右键 Bug，同时将 Day Detail 面板改为居中浮层。

**变更**：

1. `Calendar.tsx` — 修复右键 Bug：
   - Line 753 `onMouseDown` 增加 `e.button !== 2` 判断：右键（button=2）不清空 `dragSelection` 和 `tbCreateMenu`
   - 这样框选完成后，右键点击时 `dragSelection` 保持有效，`handleTimedSectionContextMenu` 正常弹出菜单

2. `Calendar.module.css` — `.detailPanel` 改为居中浮层：
   - 去掉 `top:0; right:0; bottom:0; width:400px; border-left; animation:slideInRight`
   - 改为 `top:50%; left:50%; transform:translate(-50%,-50%); width:600px; max-height:80vh; overflow-y:auto; border-radius:16px; animation:fadeScaleIn`
   - 新增 `@keyframes fadeScaleIn` 动画（从 scale(0.96) + opacity:0 到 scale(1) + opacity:1）

**验收标准**：
- [ ] 编辑模式下框选区域后，右键弹出"添加 Time Block"菜单
- [ ] Day Detail 居中显示，带背景模糊遮罩，宽度 600px，内容多时可滚动
- [ ] Day Detail 点击遮罩或 X 按钮关闭

---

### Step 2：类型即名字 + 去掉 label（TB-P2）

**目标**：去掉 TB 的独立 label 字段，type 直接作为显示名称。

**变更**：

1. `shared/types/index.ts` — `TimeBlockType` enum 改造：
   - 保留 `Study = 'study'`、`Sleep = 'sleep'` 作为内置类型
   - 删除 `Custom = 'custom'`——所有非内置类型都是具体的自定义类型名（如 "exercise"、"entertainment"、"rest"）
   - 将 `type` 字段类型从 `TimeBlockType` 改为 `string`（在 `TimeBlock`、`ResolvedTimeBlock`、`CreateTimeBlockRequest`、`UpdateTimeBlockRequest` 等接口中）

2. `Calendar.tsx` — 显示逻辑：
   - TB 背景层的 `title` 和 `<span className={styles.tbLabel}>` 直接显示 `block.type`（首字母大写）
   - 去掉所有 `block.label` 引用，改为 `block.type`
   - `handleTBCreateSubmit` 中去掉 `label` 必填校验（不再需要 `tbCreateLabel.trim()`）

3. `Calendar.tsx` — 创建/编辑表单：
   - 去掉 Label 输入框（`tbCreateLabel` / `tbEditLabel` 状态变量保留但不再使用，或直接清理）
   - 创建时 `label` 字段自动设为 type 值（兼容 DB 的 NOT NULL 约束）

4. 后端 `timeBlocks.ts` — 创建/更新路由：
   - 创建 Block 时如果没传 `label`，自动用 `type` 填充
   - 验证 `type` 不为空

5. `scheduling.ts` — Agent 排期：
   - `type === 'study'` 的判断不变（Study 仍是特殊类型）
   - 其他类型一视同仁作为"非学习时间"扣减

**验收标准**：
- [ ] TB 上显示的文字是类型名（如 "Study"、"Sleep"），不再有独立 label
- [ ] 创建表单中没有 Label 输入框
- [ ] 编辑表单中没有 Label 输入框
- [ ] DB 中 `label` 列自动填充为 type 值（向后兼容）

### Step 3：开放类型系统（TB-P3）

**目标**：将类型选择器从固定下拉改为可输入的 combobox，支持内置预设和用户自定义。

**变更**：

1. `Calendar.tsx` — 创建/编辑表单类型选择器重构：
   - 内置预设列表：`study`、`sleep`、`exercise`、`entertainment`、`rest`、`meal`
   - 选择器改为 combobox 交互：
     - 展示预设选项列表（点击选中）
     - 底部提供输入框允许用户输入自定义类型名
     - 选中后类型名显示在输入框中
   - 用户的自定义类型从已有 TB 数据中自动提取（`Set` 去重），追加到预设列表之后
   - 每个预设选项显示对应的默认颜色色块

2. `shared/types/index.ts` — 更新类型定义：
   - `TimeBlockType` enum 保留作为内置预设的引用，但接口中 `type` 字段改为 `string`
   - 新增 `TB_PRESET_TYPES` 常量数组，包含内置预设列表和对应默认颜色

3. `Calendar.tsx` / `Calendar.module.css` — Combobox 样式：
   - 下拉列表样式：每项显示颜色色块 + 类型名
   - 底部"自定义"输入行：输入框 + 确认按钮
   - 选中状态高亮

4. 单层约束更新：
   - `study` 类型保持每天最多 1 个的约束
   - 其他所有类型（内置预设 + 自定义）不限数量

**验收标准**：
- [ ] 创建表单类型选择器显示内置预设列表（study / sleep / exercise / entertainment / rest / meal）
- [ ] 用户可以从预设中选择，也可以输入自定义类型名
- [ ] 自定义类型在后续创建时自动出现在列表中（从现有 TB 数据提取）
- [ ] 编辑表单同样支持类型选择器
- [ ] Study 类型保持每天最多 1 个约束

### Step 4：优先级 Badge（TB-P4）

**目标**：TB badge 从显示总任务数改为分类显示 must / recommended / optional 计数。

**变更**：

1. `Calendar.tsx` — badge 数据计算：
   - 当前 `blockTaskCounts` 只统计总数（`Map<string, number>`）
   - 改为 `blockTaskPriorityCounts`（`Map<string, { must: number; recommended: number; optional: number }>`）
   - 遍历日任务时按 `task.priority` 分类累加

2. `Calendar.tsx` — badge 渲染：
   - 当前：`<span className={styles.tbTaskBadge}>{count} task(s)</span>`
   - 改为：三个小标签，分别用 must（红/橙色）、recommended（蓝色）、optional（灰色）对应颜色
   - 格式：紧凑型 badge，如 `2M · 1R · 3O`（或用小圆点+数字）
   - 只显示数量 > 0 的分类
   - 无任务的 Block 不显示 badge

3. `Calendar.module.css` — badge 样式：
   - `.tbTaskBadge` 拆分为 `.tbPriorityBadge` 容器 + `.tbPriorityDot` 子元素
   - 每个 dot 用对应优先级颜色（复用 `PRIORITY_COLORS` 常量）
   - 字体缩小，紧凑排列，不影响 TB 区块整体布局

**验收标准**：
- [ ] TB 上显示 must / recommended / optional 分类计数 badge
- [ ] badge 颜色与优先级对应（must=红/橙、recommended=蓝、optional=灰）
- [ ] 无任务的类别不显示（如只有 must 任务，则只显示 "2M"）
- [ ] 无任何任务的 Block 不显示 badge

## 五、技术细节

### 右键 Bug 根因

```
// 当前代码（Line 753）
onMouseDown={(e) => { setDragSelection(null); setTBCreateMenu(null); handleTBMouseDown(dayIdx, e); }}

// 问题：右键（button=2）也触发 mousedown，清空了 dragSelection
// 修复：
onMouseDown={(e) => {
  if (e.button === 2) return;  // 右键不清空选区
  setDragSelection(null);
  setTBCreateMenu(null);
  handleTBMouseDown(dayIdx, e);
}}
```

## 六、风险与约束

| 风险 | 应对 |
|------|------|
| type 从 enum 改 string 可能影响后端校验 | 后端只需确认 type 非空即可，不做枚举校验 |
| 自定义类型名可能输入不规范（大小写混乱、空格） | 前端做 trim + lowercase 标准化 |
| DB 中 label 列 NOT NULL | 保留列，创建时自动填充 type 值，不做 migration 删列 |
| 已有 TB 数据的 label 和 type 不一致 | 显示优先用 type；不做数据迁移，保持向后兼容 |
