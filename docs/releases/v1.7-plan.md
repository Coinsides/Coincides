# Coincides v1.7 — 详细实施计划

> **主题**：Task-Card 联动 — 任务-卡片多对多关联 + 查看/编辑双模式 + Card 气泡  
> **前置版本**：v1.6（Course 中心化 + UI 结构重组）  
> **目标**：打通 Task ↔ Card 双向关联，建立查看/编辑双模式交互，在查看模式中展示关联卡片气泡，完成 Course → Goal → Task → Card 知识图谱链路的最后一环。

---

## 一、设计宪法守则

| 宪法条款 | v1.7 约束 |
|---|---|
| 不替用户做决定 | Card-Task 关联由用户手动或 Agent proposal 提出，用户审批；Agent `link_task_cards` 仅在 proposal 流程中使用 |
| 不监控用户 | 不追踪卡片复习情况来自动调整任务；不根据关联数量生成未请求的分析 |
| 不制造挫败感 | TaskViewModal 不显示关联完成率统计；未关联 Card 的任务不标红/不警告 |

## 三、Step 1：task_cards 数据模型 + CRUD API + Agent 工具

### 3.1 目标

建立 Task ↔ Card 多对多关联的数据基础：新增关联表、提供 REST API、Agent 具备关联工具。

### 3.2 文件变更清单

| 文件 | 操作 | 变更内容 |
|------|------|----------|
| `server/src/db/migrations/011_task_cards.ts` | **新增** | 创建 `task_cards` 表：`id`, `task_id`, `card_id`, `checklist_index`, `created_at`；索引 `idx_task_cards_task`, `idx_task_cards_card`；UNIQUE 约束 `(task_id, card_id, checklist_index)` |
| `server/src/db/init.ts` | **修改** | `migrations` 数组追加 `011_task_cards` |
| `server/src/routes/tasks.ts` | **修改** | 新增 4 个端点：`GET /api/tasks/:taskId/cards`（获取关联 Card 列表含 checklist_index）、`POST /api/tasks/:taskId/cards`（新增关联）、`DELETE /api/tasks/:taskId/cards/:cardId`（删除关联）、`GET` 已有端点返回时附带 `linked_cards_count` 字段 |
| `server/src/routes/cards.ts` | **修改** | 新增 `GET /api/cards/:cardId/tasks`（反向查询：Card 关联的所有 Task） |
| `server/src/agent/tools/definitions.ts` | **修改** | 新增 `link_task_cards` 工具定义：`{ task_id: string, links: [{ card_id: string, checklist_index?: number }] }` |
| `server/src/agent/tools/executor.ts` | **修改** | 新增 `link_task_cards` case：批量 INSERT INTO task_cards，返回关联数量 |

### 3.3 数据模型

```sql
CREATE TABLE IF NOT EXISTS task_cards (
  id TEXT PRIMARY KEY,
  task_id TEXT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
  card_id TEXT NOT NULL REFERENCES cards(id) ON DELETE CASCADE,
  checklist_index INTEGER,  -- NULL = 任务级关联；0/1/2... = checklist item 级关联
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  UNIQUE(task_id, card_id, checklist_index)
);

CREATE INDEX IF NOT EXISTS idx_task_cards_task ON task_cards(task_id);
CREATE INDEX IF NOT EXISTS idx_task_cards_card ON task_cards(card_id);
```

### 3.4 API 设计

| 端点 | 方法 | 请求体 | 响应 |
|------|------|--------|------|
| `/api/tasks/:taskId/cards` | GET | — | `[{ id, card_id, checklist_index, card: { id, title, template_type, deck_id, deck_name } }]` |
| `/api/tasks/:taskId/cards` | POST | `{ card_id: string, checklist_index?: number }` | `{ id, task_id, card_id, checklist_index }` |
| `/api/tasks/:taskId/cards/:linkId` | DELETE | — | `204 No Content` |
| `/api/cards/:cardId/tasks` | GET | — | `[{ id, task_id, checklist_index, task: { id, title, date, status } }]` |

### 4.3 交互流

```
用户点击任务 → openModal('task-view', { task }) → TaskViewModal（只读）
   ├─ Checklist 项可勾选完成（调用 updateTask 更新 checklist）
   ├─ 点击「编辑」→ openModal('task-edit', { task }) → TaskModal（编辑）
   └─ 点击「关闭」→ closeModal()

用户 Ctrl+T / "新建任务"按钮 → openModal('task-create') → TaskModal（编辑模式，空表单）
```

│ Task Title                    [编辑] [关闭]   │

### 4.5 关键设计决策

1. **Checklist 勾选可在查看模式操作**：不需要切到编辑模式才能打勾——这是最频繁的操作
2. **task-create 不变**：创建新任务仍直接进入 TaskModal（编辑模式），没有"查看空任务"的意义
3. **DailyBrief 不受影响**：DailyBrief 页面任务点击仅 toggle 完成状态，不涉及 openModal

### 6.3 Agent 协议新增内容

```
## Task-Card Linkage（任务-卡片关联）

当创建包含 checklist 的学习任务时：
1. 如果相关 Deck 中已有对应知识点的 Card，使用 link_task_cards 建立关联
2. Checklist 的每个条目尽量映射到对应的 Card（checklist_index = 条目序号）
3. 不强制——如果没有合适的 Card，checklist item 可以无关联
4. 当用户要求"学完某章节后创建卡片"时：
   - 先创建 Task（含 checklist）
   - 再创建 Card
   - 最后用 link_task_cards 关联
5. 仅在 proposal 审批通过后执行关联——不跳过用户审批
```

## 七、关键依赖与风险

| 风险 | 影响 | 缓解方案 |
|------|------|----------|
| Checklist 是 JSON 字段，checklist_index 耦合数组顺序 | 中 | v1.7 范围内 checklist 编辑不涉及重排；未来若加重排功能需同步更新 task_cards.checklist_index |
| 所有 task-edit 调用处需改为 task-view | 中 | 代码调研已确认 6 处 Calendar + 0 处 DailyBrief；Goals 和 AppLayout 保持 task-create 不变 |
| TaskViewModal 需加载关联数据（额外 API 调用） | 低 | 打开 modal 时才加载，非全局轮询 |
| Card 删除后 CardBubble 显示 | 低 | ON DELETE CASCADE 自动清理 task_cards 记录 |

## 八、与知识图谱的关系

v1.7 完成后，Coincides 知识图谱链路完整：

```
Course ──┬── Goal ── Task ──┬── Checklist Item
         │                  └── Card[] (via task_cards)  ← v1.7 新增
         ├── Card Deck ── Section ── Card
         └── Document
```

**后续可扩展**：
- Card 详情页反向展示关联 Task
- Goal 完成度：基于 Task 完成率 + Card 复习状态综合计算
- 关联推荐：Agent 智能建议 Task-Card 关联（但需用户确认——宪法约束）
