# Coincides v1.4 — 版本规划

> **主题**：复习体验升级 + 错题本 + 成就系统 + Statistics 深度洞察
> **前置版本**：v1.3（工作流引擎 + Time Block + L1 入驻流 + 设计宪法审计）
> **目标**：将闪卡复习从"翻面评分"单一模式升级为多样化复习体验（填空题 + 选择题），引入错题本帮助用户聚焦薄弱点，建立成就系统追踪「计划 → 执行」闭环，强化 Statistics 洞察能力，并将 Goal 依赖 UI 从线性链升级为多依赖 DAG 可视化。

---

## 一、核心概念对齐

v1.3 完成了工作流引擎 + Time Block + L1 入驻流，用户已经可以从注册到计划落地日历全链路自动化。但以下环节仍然薄弱：

- **复习体验单一**：当前复习只有"翻面评分"一种模式（CardFlip → 点击翻转 → 1-4 评分）。学习科学研究表明，多模态复习（填空、选择、自由回忆）比单一模态更有效——testing effect 的强度与检索难度正相关（Bjork, 1994）。但复习模式的选择权属于用户，系统不强制。
- **错题本缺失**：用户评分 1（Again）或 2（Hard）的卡片没有归集入口。用户想聚焦薄弱点时，只能手动翻找。错题本 = 被动记录 + 用户主动查看，不强推。
- **成就系统不存在**：v1.3 的工作流引擎已追踪「计划 → 执行」闭环数据（Task 创建/完成），但没有对这些数据做正向反馈。成就系统利用已有数据，为用户提供可选的里程碑确认——**不制造挫败感**，只在用户达成时展示，不追踪失败。
- **Statistics 深度不足**：当前 Statistics 展示 streak/热力图/趋势/课程统计，但缺乏复习相关洞察（记忆保持率、卡片难度分布、复习效率）和学习节奏分析。这些数据用户点击才显示，符合设计宪法第二条。
- **Goal 依赖仅线性链**：v1.3 数据库已支持 DAG 多依赖，但 UI 限制每个 Goal 只能选一个前置。v1.4 解锁多依赖 UI + Agent 逻辑。

> v1.4 完成后，复习体验从单一翻面升级为多模态，用户可主动查看错题本聚焦薄弱点，成就系统提供正向里程碑确认，Statistics 展示复习和学习深度洞察，Goal 依赖 UI 完整支持 DAG。

## 二、设计宪法守则（v1.4 特别提醒）

v1.4 涉及成就系统和 Statistics 洞察，这两个功能最容易触碰设计宪法红线。在此重申三条原则在本版本的具体约束：

| 宪法条款 | v1.4 约束 |
|---|---|
| 不替用户做决定 | 复习模式由用户选择，系统不自动切换；错题本只记录不推送；成就不触发自动行为 |
| 不监控用户 | 不追踪复习用时（不记录每张卡的答题秒数）；Statistics 数据全部被动展示（用户点击才加载） |
| 不制造挫败感 | 成就系统只记录正向里程碑（"已完成 X"），**不记录失败**（"连续 Y 天未学"）；错题本标题用中性词（"需要加强的卡片"而非"你做错的卡片"）；Statistics 不展示"未完成率"或负面对比 |

## 四、数据模型设计

### 新增表：`wrong_book_entries`（错题本）

```sql
CREATE TABLE IF NOT EXISTS wrong_book_entries (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  card_id TEXT NOT NULL REFERENCES cards(id) ON DELETE CASCADE,
  deck_id TEXT NOT NULL,                   -- 冗余字段，便于分组查询
  course_id TEXT NOT NULL,                 -- 冗余字段，便于分组查询
  added_at TEXT NOT NULL DEFAULT (datetime('now')),
  consecutive_good INTEGER NOT NULL DEFAULT 0,  -- 连续 Good+ 次数，≥2 自动移出
  removed_at TEXT,                          -- NULL = 仍在错题本；非 NULL = 已移出
  UNIQUE(user_id, card_id)                  -- 同一卡片只有一条记录
);

CREATE INDEX IF NOT EXISTS idx_wrong_book_user ON wrong_book_entries(user_id, removed_at);
CREATE INDEX IF NOT EXISTS idx_wrong_book_deck ON wrong_book_entries(user_id, deck_id, removed_at);
```

### 新增表：`achievements`（成就定义）

```sql
CREATE TABLE IF NOT EXISTS achievements (
  id TEXT PRIMARY KEY,
  key TEXT NOT NULL UNIQUE,                -- 'first_review', 'cards_100', 'goals_5' 等
  category TEXT NOT NULL,                  -- 'review' | 'task' | 'goal' | 'general'
  tier INTEGER NOT NULL DEFAULT 1,         -- 等级（同一 key 可有多个 tier）
  threshold INTEGER NOT NULL,              -- 解锁条件阈值
  icon TEXT NOT NULL,                      -- emoji 或图标名
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### 新增表：`user_achievements`（用户解锁记录）

```sql
CREATE TABLE IF NOT EXISTS user_achievements (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  achievement_id TEXT NOT NULL REFERENCES achievements(id) ON DELETE CASCADE,
  unlocked_at TEXT NOT NULL DEFAULT (datetime('now')),
  seen INTEGER NOT NULL DEFAULT 0,         -- 用户是否已看到（控制 Toast 弹出）
  UNIQUE(user_id, achievement_id)
);

CREATE INDEX IF NOT EXISTS idx_user_ach_user ON user_achievements(user_id);
```

### 新增表：`review_sessions`（复习会话记录——用于 Statistics 复习洞察）

```sql
CREATE TABLE IF NOT EXISTS review_sessions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  started_at TEXT NOT NULL DEFAULT (datetime('now')),
  ended_at TEXT,
  mode TEXT NOT NULL DEFAULT 'flip',       -- 'flip' | 'fill_blank' | 'choice' | 'mixed'
  cards_reviewed INTEGER NOT NULL DEFAULT 0,
  cards_correct INTEGER NOT NULL DEFAULT 0  -- 评分 ≥ 3 的卡片数
);

CREATE INDEX IF NOT EXISTS idx_review_sess_user ON review_sessions(user_id);
```

### 设计说明

**错题本**：
- `wrong_book_entries` 不存储错误详情（答错了什么），只标记哪些卡片需要加强——避免"回顾失败"（设计宪法第三条）
- `consecutive_good` 计数器：每次评分 ≥ 3 时 +1，评分 < 3 时重置为 0；≥ 2 时自动将 `removed_at` 设为当前时间
- 用户手动移除 = 直接更新 `removed_at`，不影响 `consecutive_good`（用户永远有最终决定权）
- 卡片被删除时级联删除对应记录

**成就系统**：
- `achievements` 表预置种子数据（Migration 中 INSERT），不需要 Admin 后台
- 只包含里程碑型成就，例如：
  - 复习类：完成第 1 次复习、累计复习 50/100/500 张卡片
  - 任务类：完成第 1 个 Task、累计完成 10/50/100 个 Task
  - 目标类：完成第 1 个 Goal、累计完成 5/10 个 Goal
  - 综合类：创建第 1 门课程、上传第 1 份文档
- **不包含**：streak 类（"连续 X 天"）、完成率类（"本周完成 100%"）、时间类（"学习 X 小时"）
- 检测时机：已有代码路径中的事件触发（rateCard → 检查复习成就、completeTask → 检查任务成就、completeGoal → 检查目标成就），不新增定时任务

**复习会话**：
- `review_sessions` 只记录会话级聚合数据（多少张卡、正确率），**不记录每张卡的答题时间**——设计宪法第二条
- 用于 Statistics 复习洞察面板（复习量趋势、模式使用分布）

**填空/选择题**：
- 不新增表——复习模式是运行时行为，不改变卡片数据结构
- 填空题：基于 Card 的 `content` 字段，在前端动态遮掩关键词（按模板类型有不同策略）
- 选择题：运行时从同 Deck/Section 的其他卡片中抽取干扰项，不预存

## 五、Step 拆分

### Step 1：错题本数据模型 + API + 复习评分集成（WB-1 ~ WB-5）

**目标**：建立错题本后端基础，评分流程中自动标记/移出错题本。

**工作内容**：

**数据模型**：
- Migration `009_wrong_book.ts`：创建 `wrong_book_entries` 表
- `shared/types/index.ts`：新增 `WrongBookEntry` 接口

**API**：
- `server/src/routes/wrongBook.ts`：
  - `GET /api/wrong-book` — 获取用户错题本（支持 `course_id`、`deck_id` 筛选）
  - `GET /api/wrong-book/count` — 获取错题本卡片总数
  - `DELETE /api/wrong-book/:cardId` — 手动移除（用户决定权）
- 评分集成：修改 `server/src/routes/review.ts` 中的 `POST /review/:cardId/rate`：
  - 评分 = 1（Again）→ upsert 到 `wrong_book_entries`（reset `consecutive_good` 为 0）
  - 评分 ≥ 3 → 如果存在记录，`consecutive_good` +1；如果 ≥ 2，设置 `removed_at`

**Agent 工具**：
- 新增 `get_weak_cards`：返回用户错题本卡片列表（含 deck_name、course_name），Agent 可据此建议复习

**验收标准**：
- [ ] 评分为 1 的卡片自动进入错题本
- [ ] 连续 2 次评分 ≥ 3 自动移出错题本
- [ ] 手动移除正常工作
- [ ] 错题本 API 支持按课程/Deck 筛选
- [ ] Agent `get_weak_cards` 返回正确数据
- [ ] 卡片删除时级联清理错题本记录

### Step 2：复习模式引擎——填空题 + 选择题（RV-1 ~ RV-5）

**目标**：在翻面评分基础上，新增填空题和选择题两种复习模式，用户可选择。

**工作内容**：

**复习模式选择 UI**：
- Review 页面开始复习前，新增模式选择面板（卡片式布局，复用 Glassmorphism 风格）：
  - 📖 翻面评分（默认）— "经典的翻卡复习"
  - ✏️ 填空题 — "遮掩关键内容，测试回忆"
  - 🔢 选择题 — "从选项中选出正确答案"
  - 🎲 混合模式 — "随机切换模式，增加挑战"
- Settings 页面新增"默认复习模式"偏好

**填空题引擎**（RV-2）：
- `client/src/components/ReviewModes/FillBlankMode.tsx`：
  - 根据卡片模板类型遮掩关键内容：
    - Definition：遮掩 `definition` 字段的关键名词
    - Theorem：遮掩 `statement` 字段的核心结论部分
    - Formula：遮掩 `formula` 字段（最适合填空）
    - General：遮掩 `back` 字段的第一句
  - 遮掩策略：用 `____` 替换，KaTeX 公式中遮掩变量/结果
  - 用户输入答案 → 显示对比（绿色正确/橙色接近/红色不同）→ 自行评分（1-4）
  - **不自动判分**——用户自己决定掌握程度（宪法第一条）

**选择题引擎**（RV-3）：
- `client/src/components/ReviewModes/ChoiceMode.tsx`：
  - 问题 = 卡片 front（title + 适当提示）
  - 正确答案 = 卡片 back 内容摘要
  - 干扰项生成：从同 Deck/Section 的其他卡片中随机选取 3 张的 back 内容
  - 4 个选项随机排列
  - 用户选择后 → 高亮正确答案（绿色）+ 用户选择（红色/绿色）→ 自行评分（1-4）
  - 如果同 Deck 卡片不足 4 张，回退到翻面模式（静默回退，不提示用户）

**混合模式**（RV-4）：
- 复习会话开始时，为每张卡片随机分配模式（flip/fill_blank/choice）
- 会话中无缝切换，卡片间使用不同的复习组件

**复习会话记录**：
- Migration `010_review_sessions.ts`：创建 `review_sessions` 表
- 复习会话开始时创建记录，结束时更新 `ended_at` + `cards_reviewed` + `cards_correct`

**i18n**：所有新增文案添加中英翻译

**验收标准**：
- [ ] 复习前可选择 4 种模式
- [ ] 填空题正确遮掩各模板类型的关键内容
- [ ] KaTeX 公式在填空模式下正确渲染遮掩效果
- [ ] 选择题干扰项来自同 Deck 其他卡片
- [ ] 卡片不足时静默回退到翻面模式
- [ ] 混合模式随机切换工作正常
- [ ] 复习会话记录正确保存
- [ ] Settings 中可设置默认模式
- [ ] 中英双语正常

## 十、风险与阻塞项

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 填空题遮掩不够智能 | 遮掩了无关紧要的词，或没遮掩关键内容 | v1.4 用基于模板类型的规则遮掩（确定性策略），不用 AI 实时判断；后续版本可引入 AI 智能遮掩 |
| 选择题干扰项质量差 | 干扰项和正确答案差距太大，用户一眼看出 | 优先从同 Section 选（内容更接近），其次同 Deck；未来可用 embedding 相似度选最接近的干扰项 |
| KaTeX 公式遮掩复杂度 | 公式中遮掩变量可能破坏 LaTeX 语法 | 公式类卡片默认遮掩整个 formula 字段（而非内部变量），保证渲染不崩 |
| 成就检测性能 | 每次评分/完成任务都查数据库 | 成就数量有限（~14 个），SQL 查询轻量（COUNT + JOIN），可接受 |
| Statistics SVG 图表复杂度 | 纯 SVG 绘制多图表工作量大 | 复用现有 TrendChart 组件模式，提取公共 SVG 工具函数 |
| DAG 可视化交叉线 | 复杂 DAG 箭头交叉不美观 | 限制显示深度（2 层），超过则切换列表视图；后续可引入自动布局算法 |
| 动态优先级对用户不透明 | 用户不理解为什么排期变了 | Agent 在排期回复中用自然语言解释："因为 XX 考试临近，相关任务优先安排" |
| review_sessions 与 FSRS 的关系 | 新增的 review_sessions 是否影响 FSRS 评分流程？ | 完全不影响——review_sessions 是独立的统计表，FSRS 评分流程（rateCard → ts-fsrs）不变 |
