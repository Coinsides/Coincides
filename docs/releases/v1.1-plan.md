# Coincides v1.1 — 版本规划

> **主题**：UX 痛点修复 + 技术债偿还 + 数据模型升级
> **前置版本**：v1.0（Round 4 完成）
> **目标**：修复 6 个 UX 痛点，偿还 4 项技术债，同时升级 Task 和 Goal 的数据模型为后续版本打基础。

---

## 一、核心概念对齐

本次会议确认的数据模型定义：

- **Task = 日历事件**：有具体时间段（start_time / end_time）、优先级（Must / Recommended / Optional）、checklist、描述。排在日历上就是用户看到的事件。
- **Goal = 任务的聚合**：代表一个阶段性成果（如"学完 Chapter 1-3"）。Goal 支持嵌套——大目标包含子目标，子目标下挂 Task。
- **工作流**：用户先在 Goal Manager 中创建目标 → 拆分阶段（子 Goal）→ 在每个阶段下创建 Task → 排期到日历。

> v1.1 先做数据模型升级（schema）和基础 UI，完整的 Goal Manager UI 和 Agent 集成留给 v1.2/v1.3。

## 三、Step 拆分

### Step 1：DB Migration 机制 + 配置验证（T-3 + T-4）

**目标**：建立 migration 基础设施，所有后续 schema 变更通过 migration 执行。

**工作内容**：
- `server/src/db/migrations/` 目录 + migration runner
- `db_version` 表记录当前 schema 版本
- 服务器启动时自动检测并执行未运行的 migration
- 启动时验证必需环境变量（ANTHROPIC_API_KEY），缺失则报错退出
- 可选变量（VOYAGE_API_KEY）缺失打警告但继续运行

**验收标准**：
- [ ] migration 目录存在，runner 可用
- [ ] 服务器启动时自动执行未运行的 migration，不重复执行
- [ ] 缺少 ANTHROPIC_API_KEY → 明确报错并退出
- [ ] 缺少 VOYAGE_API_KEY → 打警告，正常启动

---

### Step 2：Task 数据模型升级（M-1）

**目标**：将 Task 升级为完整的日历事件，具备时间段、优先级、描述和 checklist。

**Schema 变更（通过 migration）**：
```sql
ALTER TABLE tasks ADD COLUMN start_time TEXT;      -- ISO 8601
ALTER TABLE tasks ADD COLUMN end_time TEXT;         -- ISO 8601
ALTER TABLE tasks ADD COLUMN priority TEXT DEFAULT 'must';  -- must/recommended/optional
ALTER TABLE tasks ADD COLUMN description TEXT;
ALTER TABLE tasks ADD COLUMN checklist TEXT;        -- JSON array: [{text, done}]
```

**工作内容**：
- Migration 脚本：给 tasks 表加 5 个字段
- 后端 API 更新：tasks CRUD 支持新字段
- 前端表单更新：创建/编辑 Task 时可填写时间段、优先级、描述
- 现有数据兼容：新字段全部 nullable 或有默认值，不影响已有 Task

**验收标准**：
- [ ] Migration 自动执行，现有 tasks 数据不丢失
- [ ] 新建 Task 时可选择 start_time / end_time / priority
- [ ] 已有 Task 的 priority 默认为 must
- [ ] checklist 以 JSON 数组存储，前端可增删勾选

---

### Step 3：Goal 嵌套支持（M-2 + G-1）

**目标**：Goal 支持父子关系，允许随时向 Goal 添加任务。

**Schema 变更（通过 migration）**：
```sql
ALTER TABLE goals ADD COLUMN parent_id INTEGER REFERENCES goals(id) ON DELETE CASCADE;
```

**工作内容**：
- Migration 脚本：goals 表加 parent_id
- 后端 API 更新：
  - `POST /api/goals` 支持 parent_id 参数
  - `GET /api/goals` 返回层级结构（或通过 parent_id 过滤）
  - `POST /api/goals/:id/tasks` 向已有 Goal 添加任务（修复 G-1）
- 前端 Goals 页面：
  - 显示 Goal 层级（大目标 → 子目标）—— v1.1 先做一层缩进展示
  - Goal 详情页新增「添加任务」按钮
  - Goal 详情页新增「添加子目标」按钮
- 完整的 Goal Manager UI（树形拖拽、阶段管理）留给 v1.2

**验收标准**：
- [ ] 可创建子 Goal（指定 parent_id）
- [ ] Goals 页面按层级展示
- [ ] 已有 Goal 可随时添加新 Task（G-1 修复）
- [ ] 删除父 Goal 时子 Goal 级联删除

---

### Step 4：日历 CRUD 补全（C-1 + C-2 + G-2）

**目标**：日历视图完整支持 Task 的查看、编辑、删除，并按优先级分层展示。

**工作内容**：
- 日历渲染升级：使用 Task 的 start_time / end_time 渲染为时间块
- 右键/长按菜单：编辑、删除
- 编辑弹窗：修改名称、时间段、优先级、描述
- DailyBrief 按 Must / Recommended / Optional 分层展示（G-2）
- 无时间段的 Task（旧数据）仍以全天事件形式展示

**验收标准**：
- [ ] 有时间段的 Task 在日历上显示为时间块
- [ ] 右键/长按弹出菜单，支持编辑和删除（C-1, C-2）
- [ ] 编辑保存后日历即时更新
- [ ] DailyBrief 按三级优先级分层展示（G-2）
- [ ] 旧的无时间段 Task 正常显示，不报错

### Step 5：跨 Deck 复习选择器（R-1）

**验收标准**：
- [ ] 可从 Review 页面进入自定义复习
- [ ] 可跨 Deck / Section 勾选任意卡片
- [ ] 显示已选数量，点击「开始复习」进入复习流程
- [ ] 复习流程与现有一致（翻转、评分、FSRS 更新）

## 五、为后续版本铺路

v1.1 的数据模型升级为后续版本打基础：

| v1.1 做的 | 为哪个版本铺路 | 后续怎么用 |
|---|---|---|
| Task 加时间段 | v1.3 Time Block | Time Block = 周级 Task 模板，复用同一套字段 |
| Task 加 priority | v1.2 AI 交互 | Agent 生成 Proposal 时自动标注优先级 |
| Task 加 checklist | v1.2 AI 交互 | Agent 可拆解任务为子步骤 |
| Goal 嵌套 | v1.2 Goal Manager UI | 完整的树形展示 + AI 自动拆阶段 |
| Goal 嵌套 | v1.3 工作流引擎 | L1 入驻流：大目标 → 阶段 → 排期 |
